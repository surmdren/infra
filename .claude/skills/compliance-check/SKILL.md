# 法律合规文档生成器

## Overview

根据技术方案生成结构化的**法律合规评估文档模板**，提供检查清单和填写框架。

> **注意**：本 skill 只生成文档**模板**，不做合规性审核判断。具体风险评估由法务人员或项目负责人填写。

```
1. 读取技术方案/项目信息
2. 生成合规性评估文档模板
3. 输出结构化的检查清单
4. 用户/法务填写具体内容
```

## 输出文档结构

```
Compliance-Report/
├── README.md           # 合规评估总览
├── open-source.md      # 开源协议清单
├── data-privacy.md     # 数据隐私合规
├── dependencies.md     # 第三方依赖清单
├── ip-rights.md        # 知识产权检查
└── action-items.md     # 后续行动清单
```

## 文档模板

完整的合规评估文档模板（主文档 README.md 及各子文档）见 `references/compliance-template.md`。

模板涵盖以下维度：

| 维度 | 检查内容 |
|------|---------|
| 开源协议 | 核心依赖协议清单、协议风险说明（MIT/Apache/GPL/AGPL）、待确认事项 |
| 数据隐私 | 数据收集清单、《个人信息保护法》/《数据安全法》/GDPR 对照、合规文件状态 |
| 第三方依赖 | SDK/服务清单、CVE 安全漏洞检查 |
| 知识产权 | 商标检查、专利风险、著作权（代码/图片/字体/音视频） |
| 风险结论 | 四维度风险汇总、必须/建议处理事项 |
| 后续行动 | 文档准备、技术措施、法律咨询、时间计划 |

## 使用方式

### 方式 1: 基于技术方案生成

```
用户: 根据这个技术方案生成合规文档模板

Claude:
1. 读取技术方案
2. 提取关键信息（技术栈、依赖、数据处理等）
3. 生成合规评估文档模板
4. 用户填写具体内容
```

### 方式 2: 生成空白模板

```
用户: 生成一份法律合规评估文档模板

Claude:
1. 生成完整的合规评估文档框架
2. 包含所有检查清单和表格
3. 用户/法务逐项填写
```

## 输出格式

| 参数 | 说明 |
|------|------|
| `{project_name}` | 项目名称 |
| `{date}` | 生成日期 |
| `{tech_stack}` | 技术栈（从方案中提取） |
| `{count}` | 依赖数量（从配置中统计） |

## 填写指南

### 风险等级定义

| 等级 | 说明 |
|------|------|
| 低 | 无明显风险，可正常商用 |
| 中 | 存在风险，需注意或处理 |
| 高 | 重大风险，必须处理 |

### 开源协议判断

| 特征 | 判断依据 |
|------|----------|
| 可商用 | MIT/Apache/BSD 协议 |
| 需注意 | LGPL - 动态链接可闭源，静态链接需开源 |
| 高风险 | GPL/AGPL - 衍生作品需开源 |
| 需确认 | 自定义协议、双重许可 |
