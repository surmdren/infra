# PIPL vs GDPR 对照表

适用于商用软件上线前的隐私合规自查。

---

## 核心要求对照

| 维度 | PIPL（个人信息保护法，中国） | GDPR（欧盟） |
|------|--------------------------|------------|
| 适用范围 | 在中国境内处理个人信息 | 处理欧盟居民数据 |
| 生效时间 | 2021年11月1日 | 2018年5月25日 |
| 个人信息定义 | 与已识别或可识别自然人相关的各种信息 | 与已识别或可识别自然人相关的任何信息 |
| 处理的合法基础 | 同意、合同、法定义务、公共利益、保护重大利益 | 同意、合同、法定义务、重大利益、公共任务、合法利益 |
| 用户权利 | 知情权、决定权、查询权、复制权、更正权、删除权 | 访问权、更正权、删除权（被遗忘权）、限制处理权、数据可携权、拒绝权 |
| 数据本地化 | 关键信息基础设施运营者须在境内存储；跨境传输需评估 | 无强制本地化，但跨境传输有限制 |
| 处罚力度 | 最高 5000 万元或年营业额 5% | 最高 2000 万欧元或年营业额 4% |
| 隐私政策 | 必须提供，需告知处理目的、方式、范围 | 必须提供（隐私通知），需符合透明度要求 |

---

## 商用软件必须满足的基本要求（PIPL）

### 1. 收集前取得用户同意
- 用户注册/使用前必须展示隐私政策
- 同意必须是主动的、明确的（不能预勾选）
- 敏感个人信息（生物识别、医疗健康、金融账户）需**单独同意**

### 2. 最小化原则
- 只收集完成目的所必需的数据
- 不能收集与服务无关的信息（如天气 App 索要通讯录）

### 3. 数据保留
- 明确告知数据保留期限
- 超过保留期或达到目的后，及时删除或匿名化

### 4. 用户权利响应
- 用户提出查询/更正/删除申请，需在**15个工作日**内响应
- 需提供账号注销功能

### 5. 数据安全
- 采取技术措施保护数据安全
- 发生数据泄露需**及时通报**主管机关及受影响用户

## 风险等级定义

### 🔴 高风险（代码层面，需立即修复）

| 问题 | PIPL 条款 | GDPR 条款 |
|------|----------|----------|
| PII 硬编码在代码中 | 第 51 条（安全保护义务） | Article 32（处理安全） |
| 密码明文存储 | 第 51 条 | Article 32 |
| 日志中记录完整手机号/身份证 | 第 51 条 | Article 32 |
| CORS 允许所有来源（可能导致数据跨站泄露） | 第 51 条 | Article 32 |
| 数据库连接字符串泄露 | 第 51 条 | Article 32 |

### 🟡 中风险（需评估和修复）

| 问题 | 说明 |
|------|------|
| Cookie 无 HttpOnly/Secure | XSS 可窃取 Session，导致账号泄露 |
| localStorage 存 Token | XSS 可读取，降级到 httpOnly Cookie |
| HTTP 明文传输 | 中间人攻击可截获数据 |
| Debug 模式开启 | 可能暴露系统内部信息 |
| Session 过期时间过长 | 增加 Token 被盗后的危害窗口 |

### 🟢 低风险（知悉，规范化处理）

| 问题 | 说明 |
|------|------|
| 内网 IP 硬编码 | 暴露内部网络结构，但不直接危害用户数据 |
| 日志级别过详细 | 生产环境改为 info/warn |
| 伪随机数用于非安全场景 | 如果用于 Token/验证码需换成 crypto 模块 |

## 上线前合规检查清单

### 基础要求
- [ ] 隐私政策页面存在且可访问
- [ ] 首次使用前展示隐私政策并获取同意
- [ ] 提供账号注销功能
- [ ] 提供用户数据导出功能（可选但推荐）

### 数据安全
- [ ] 所有 API 强制 HTTPS
- [ ] 用户密码使用 bcrypt/argon2 哈希存储
- [ ] 敏感数据（身份证、银行卡）加密存储
- [ ] 生产环境 .env 未提交到 Git
- [ ] Cookie 设置 HttpOnly + Secure + SameSite

### 数据最小化
- [ ] 登录只收集必要信息（邮箱/手机号，不强制姓名）
- [ ] 未使用第三方 SDK 的非必要权限（如位置、通讯录）
- [ ] 日志中手机号/身份证已脱敏

### 出海合规补充（GDPR）
- [ ] Cookie 同意弹窗（Cookie Banner）
- [ ] 数据处理协议（DPA）
- [ ] 数据主体请求响应机制（30天内）
