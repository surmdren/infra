# Market Research Skill

## 目标

帮助用户快速理解一个行业，并识别其中的赚钱机会（创业、投资、副业）。

## 输入支持

用户可以提供以下任意形式的资料：
- PDF 行业报告、白皮书
- 网页链接（行业新闻、公司官网、分析文章）
- 直接文本描述
- 多种形式组合

## 工作流程

### Step 1: 收集和理解资料

1. **接收用户输入**
   - 如果是文件路径，使用 Read 工具读取
   - 如果是 URL，使用 WebFetch 工具获取内容
   - 如果是文本，直接分析

2. **提取关键信息**
   参考 `references/analysis-framework.md` 中的分析维度，从资料中提取：
   - 市场规模与增长数据
   - 主要玩家和竞争格局
   - 用户需求和痛点
   - 商业模式和变现方式
   - 技术和政策趋势

3. **识别信息缺口**
   如果关键信息缺失，可以：
   - 使用 WebSearch 补充最新数据
   - 标注信息不足的部分
   - 基于现有信息做合理推断（标注为推断）

### Step 2: 分析和提炼

使用 `references/analysis-framework.md` 中的框架进行系统分析：

1. **行业分析**
   - 市场规模与增长
   - **产业链全景图（核心重点）**
     - 绘制上游-中游-下游完整结构
     - 标注关键参与者
     - 分析价值分布和利润池
     - 识别议价能力和话语权
     - 使用 Mermaid 图可视化（参考 `references/analysis-framework.md` 示例）
   - 竞争格局
   - 用户画像
   - 商业模式
   - 技术趋势
   - 政策环境

2. **机会识别**
   从三个维度寻找机会：

   **创业机会:**
   - 市场空白点（未满足的需求）
   - 效率提升空间（流程优化）
   - 体验升级机会（痛点解决）
   - 模式创新机会（新商业模式）

   **投资机会:**
   - 高增长赛道（CAGR > 20%）
   - 头部公司（护城河深）
   - 潜力新星（差异化竞争）

   **副业/自由职业:**
   - 低门槛入口（资金/技能要求低）
   - 快速变现（回报周期短）
   - 个人品牌（专业服务/内容创作）

3. **机会评估**
   对每个识别的机会，使用评估矩阵打分（1-5分）：
   - 市场规模 (20%)
   - 增长潜力 (20%)
   - 竞争强度 (15%)
   - 进入门槛 (15%)
   - 变现能力 (15%)
   - 执行难度 (10%)
   - 风险程度 (5%)

   计算综合得分，确定优先级。

### Step 3: 生成报告

1. **创建输出目录**
   ```bash
   mkdir -p market_research/{行业名称}
   ```

2. **生成主报告**
   使用 `assets/report-template.md` 作为模板，填充分析结果。

   报告结构：
   - 执行摘要（关键发现 + Top 3 机会）
   - 行业概览
     - **产业链全景图**（必需，使用Mermaid可视化）
       - 上游-中游-下游完整结构
       - 关键参与者列表
       - 价值分布与利润池分析
       - 议价能力和话语权分析
   - 竞争格局
   - 用户洞察
   - 商业模式
   - 趋势与变化
   - 赚钱机会分析（每个机会详细展开）
   - 行动建议

3. **创建机会清单**
   生成 `opportunities.md`，列出所有识别的机会及评分。

4. **保存原始资料**
   如果用户提供了 URL 或文件，在报告中记录数据来源。

### Step 4: 交付成果

输出文件到 `market_research/{行业名称}/` 目录：
- `report.md` - 完整市场研究报告
- `opportunities.md` - 机会清单（按优先级排序）
- `sources.txt` - 数据来源链接

告知用户：
- 报告位置
- Top 3 推荐机会及理由
- 建议的下一步行动

## 输出质量标准

### 必须包含
- ✅ 清晰的行业定义和范围
- ✅ 具体的数字和数据（市场规模、增长率）
- ✅ 至少 3-5 个赚钱机会，每个都有详细分析
- ✅ 每个机会的评分和优先级
- ✅ 可执行的行动建议

### 优秀实践
- 📊 使用表格展示对比数据
- 🎯 机会描述具体，避免空泛
- 💡 提供执行路径，不只是描述机会
- ⚠️ 标注风险和挑战，保持客观
- 📚 注明数据来源，提高可信度

### 避免
- ❌ 过于笼统的描述（"市场很大"→ 具体金额）
- ❌ 没有数据支撑的判断
- ❌ 机会描述模糊（"做电商"→ "针对X人群的Y品类电商"）
- ❌ 忽略竞争和风险
- ❌ 缺少可执行的下一步

## 示例用法

**示例 1: PDF 报告分析**
```
用户: 我有一份关于 AI 教育的行业报告，帮我分析一下有什么赚钱机会
[提供 report.pdf]

Claude:
1. 读取 PDF 内容
2. 分析行业现状
3. 识别 5 个机会（2个创业、2个投资、1个副业）
4. 生成报告到 market_research/ai-education/
5. 推荐 Top 3 机会
```

**示例 2: 网页资料分析**
```
用户: 帮我研究一下跨境电商行业，看看有什么机会
提供几个参考链接：
- https://example.com/cross-border-ecommerce-report
- https://example.com/market-analysis

Claude:
1. 使用 WebFetch 获取页面内容
2. 使用 WebSearch 补充最新数据
3. 综合分析
4. 生成报告到 market_research/cross-border-ecommerce/
```

**示例 3: 文本描述分析**
```
用户: 我想了解一下宠物经济，特别是宠物食品这块，有什么机会？

Claude:
1. 使用 WebSearch 获取行业数据
2. 重点分析宠物食品细分市场
3. 识别机会（创业、投资、副业）
4. 生成报告
```

## 注意事项

1. **数据时效性**: 使用 WebSearch 获取最新数据时，明确标注数据日期
2. **客观性**: 保持分析客观，不过度乐观或悲观
3. **可执行性**: 机会分析要具体，提供清晰的执行路径
4. **风险意识**: 每个机会都要评估风险，帮助用户理性决策
5. **用户导向**: 输出报告应易读易懂，避免过度专业术语
