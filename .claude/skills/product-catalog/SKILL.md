# Product Catalog - 公司产品目录整理器

整理公司产品信息，生成结构化产品知识库，便于快速学习和客户演示。

## 工作流程

### Step 1: 收集产品信息

支持多种输入方式：

**A. 公司官网 URL**
```
用户提供: https://company.com/products
使用 WebFetch 获取产品页面内容，提取产品列表和详情
```

**B. 本地文档**
```
用户提供: /path/to/产品手册.pdf
使用 Read 工具读取，解析产品信息
支持格式: PDF, Word, Excel, Markdown
```

**C. 口头/文字描述**
```
用户直接描述产品信息
例如: "我们公司主要有三大产品线：检测设备、分析软件、配套耗材..."
```

**D. 混合方式**
```
组合以上多种输入，交叉验证和补充信息
```

### Step 2: 产品分类

根据产品数量和复杂度自动选择分类层级：

**产品数 ≤ 10 → 两层结构**
```
大类 > 产品
例如:
├── 检测设备
│   ├── 产品A
│   └── 产品B
└── 分析软件
    └── 产品C
```

**产品数 > 10 → 三层结构**
```
大类 > 子类 > 产品
例如:
├── 检测设备
│   ├── 光学检测
│   │   ├── 产品A
│   │   └── 产品B
│   └── 电学检测
│       └── 产品C
└── 分析软件
    ├── 数据分析
    │   └── 产品D
    └── 图像处理
        └── 产品E
```

### Step 3: 提取产品信息

为每个产品提取以下信息：

| 字段 | 描述 | 必填 |
|------|------|------|
| 产品名称 | 中文名 + 英文名/型号 | ✅ |
| 一句话描述 | 20字以内的核心定位 | ✅ |
| 产品类别 | 所属分类路径 | ✅ |
| 核心特点 | 3-5个差异化特点 | ✅ |
| 应用场景 | 典型使用场景 | ✅ |
| 目标客户 | 主要客户群体 | ⚪ |
| 技术参数 | 关键技术指标 | ⚪ |
| 竞品对比 | 与竞品的差异点 | ⚪ |
| 价格区间 | 价格范围(如有) | ⚪ |

### Step 4: 生成输出

创建输出目录：
```bash
mkdir -p product_catalog/{公司名称}
```

#### 输出 1: Markdown 产品手册

文件: `product_catalog/{公司名称}/catalog.md`

结构:
```markdown
# {公司名称} 产品目录

> 生成时间: YYYY-MM-DD
> 产品总数: N 个

## 产品概览

[产品分类树形图]

## 产品详情

### 大类1: {类别名}

#### {产品名称} ({型号})

**一句话描述**: ...

**核心特点**:
- 特点1
- 特点2
- 特点3

**应用场景**:
- 场景1
- 场景2

**技术参数**:
| 参数 | 值 |
|------|-----|
| xxx | xxx |

---

[重复以上结构]
```

#### 输出 2: Anki 记忆卡片

文件: `product_catalog/{公司名称}/anki_cards.csv`

生成可导入 Anki 的 CSV 文件，包含以下卡片类型：

**卡片类型 1: 产品识别**
- 正面: 产品名称
- 背面: 一句话描述 + 核心特点

**卡片类型 2: 特点记忆**
- 正面: {产品名} 的核心特点是什么？
- 背面: 3-5个特点列表

**卡片类型 3: 场景应用**
- 正面: 客户需要{场景}，推荐什么产品？
- 背面: 产品名 + 理由

CSV 格式:
```csv
Front,Back,Tags
"产品A是什么？","一句话描述 + 特点","product::category"
```

#### 输出 3: PDF 速查表

文件: `product_catalog/{公司名称}/quick-reference.md` (Markdown 格式，可转PDF)

单页/双页速查表，包含：
- 产品分类一览
- 各产品核心卖点（每产品1-2行）
- 场景-产品对照表
- 关键参数对比表

### Step 5: 交付成果

输出文件清单：
```
product_catalog/{公司名称}/
├── catalog.md          # 完整产品手册
├── anki_cards.csv      # Anki 导入文件
├── quick-reference.md  # 速查表
└── sources.txt         # 信息来源
```

告知用户：
- 产品总数和分类结构
- 文件保存位置
- 建议的学习顺序
- 如需补充的信息

## 输出质量标准

### 必须包含
- ✅ 清晰的产品分类结构
- ✅ 每个产品的基本信息完整
- ✅ 一句话描述精准（易记易传播）
- ✅ 特点具体可感（避免空泛形容词）
- ✅ 三种输出格式齐全

### 优秀实践
- 📊 用表格对比同类产品
- 🎯 特点描述使用"动词+结果"格式
- 💡 提供记忆口诀或助记词
- 🔗 关联产品交叉引用

### 避免
- ❌ 特点描述空泛（"性能优异"→"处理速度提升50%"）
- ❌ 专业术语堆砌无解释
- ❌ 忽略产品间的差异对比
- ❌ 信息来源不明确

## 示例用法

**示例 1: 从官网提取**
```
用户: 帮我整理一下我们公司的产品目录
官网: https://example.com/products

Claude:
1. WebFetch 获取产品页面
2. 解析产品列表和详情页
3. 提取信息并分类
4. 生成三种格式输出
```

**示例 2: 从文档提取**
```
用户: 这是我们的产品手册，帮我整理成产品目录
文件: /docs/product-manual.pdf

Claude:
1. Read 读取 PDF
2. 提取产品信息
3. 自动分类
4. 生成输出文件
```

**示例 3: 口头描述**
```
用户: 我们公司有三个产品线：
1. 光学检测设备 - 主要有偏振检测仪和光谱分析仪
2. 软件平台 - 数据分析软件
3. 耗材配件 - 标准片、校准片

Claude:
1. 解析用户描述
2. 使用 WebSearch 补充产品详情
3. 整理并确认信息
4. 生成产品目录
```

## 与其他 Skills 配合

| 场景 | 配合 Skill | 用法 |
|------|-----------|------|
| 了解竞品 | competitor-analysis | 先分析竞品，再整理自家产品对比 |
| 销售培训 | industry-sales-prep | 产品目录 + 行业知识 = 完整销售知识库 |
| 市场定位 | market-research | 结合市场研究确定产品定位 |

## 注意事项

1. **信息保密**: 确认用户授权后再使用公开渠道补充信息
2. **信息准确**: 优先使用用户提供的官方资料，外部搜索仅作补充
3. **持续更新**: 建议用户定期更新产品目录（如每季度）
4. **格式兼容**: Anki CSV 使用 UTF-8 编码，确保中文正常显示
