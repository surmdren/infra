# PII 正则规则库

## 中国大陆常见 PII 类型

### 手机号
```
正则：1[3-9]\d{9}
```
- 覆盖：13x、14x、15x、16x、17x、18x、19x 号段
- **误报场景**：版本号（如 1.3.9.1234）、时间戳中的数字组合
- **减少误报**：需检查前后是否有其他数字，以及是否在测试代码中

### 身份证号
```
正则：[1-9]\d{5}(19|20)\d{2}(0[1-9]|1[0-2])(0[1-9]|[12]\d|3[01])\d{3}[\dXx]
```
- 18位，含校验位（X或x）
- 误报率极低，建议直接报高风险

### 银行卡号
```
正则：[3-6]\d{15,18}
```
- 覆盖 16-19 位银行卡
- **注意**：容易误报，需结合上下文（如变量名含 card、bank、account）

### 护照号（中国）
```
正则：[EeGg][0-9]{8}
```
- E 开头为普通护照，G 开头为公务护照

### 车牌号
```
正则：[京津沪渝...][A-Z][A-Z0-9]{5}
```
- 仅适用于标准车牌，新能源车牌格式不同

## 凭据类 PII

### 硬编码密码
```
正则：(?i)(password|passwd|pwd)\s*[=:]\s*['"][^'"]{6,}['"]
```
- 高可信度，基本无误报
- **例外**：注释中的示例密码（如 `# password: your_password_here`）

### API Key
```
正则：(?i)(api_key|apikey|access_key)\s*[=:]\s*['"][A-Za-z0-9+/\-_]{16,}['"]
```
- **注意**：`your_api_key_here` 这类占位符应排除

### JWT Token
```
正则：eyJ[A-Za-z0-9+/\-_]{20,}\.[A-Za-z0-9+/\-_]{20,}\.
```
- JWT 以 eyJ 开头（base64 编码的 `{"` ）
- 高风险，任何 JWT 硬编码都是问题

### AWS Access Key
```
正则：(AKIA|ASIA|AROA|AIDA)[A-Z0-9]{16}
```
- AWS 官方格式，几乎无误报

## 降低误报的原则

1. **测试文件豁免**：`test/`、`spec/`、`__tests__/` 目录中的命中可降低风险等级
2. **注释行豁免**：`//`、`#`、`/*` 开头的行通常是示例，可降低风险等级
3. **占位符豁免**：含 `your_`、`<your`、`xxx`、`change_me`、`placeholder` 的值可跳过
4. **长度过滤**：密码类规则要求最少 6 个字符，避免匹配变量名

## 脱敏建议

发现 PII 后的推荐脱敏格式：

| 类型 | 脱敏示例 |
|------|---------|
| 手机号 | `138****1234` |
| 身份证 | `330102****0012` |
| 银行卡 | `**** **** **** 1234` |
| 邮箱 | `u***@example.com` |
| 姓名 | `张*` |

## 日志脱敏模式（代码片段参考）

```python
# Python
import re
def mask_phone(phone):
    return re.sub(r'(\d{3})\d{4}(\d{4})', r'\1****\2', phone)
```

```javascript
// JavaScript/TypeScript
const maskPhone = (phone) => phone.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2');
```
