# Oral Style Guide — DreamWise AI YouTube Channel

> 适用范围：所有视频脚本生成（video-script-generator）
> 参考样本：`202603/001/script-oral.md`
> 核心原则：**听起来像人说的，不像人写的**

---

## 一、句子结构

**极短句为主，一个想法一行。**

```
✅ Done.
✅ Same issue.
✅ Over and over.
✅ Why?

❌ The process is now complete and the same error will occur again.
```

**列举用换行，不用逗号堆叠：**

```
✅ It controls:
   how tasks are executed
   how tools are used
   how errors are handled

❌ It controls how tasks are executed, how tools are used, how errors are handled.
```

---

## 二、停顿与节奏标记

| 标记 | 含义 | 示例 |
|------|------|------|
| `…` | 口头停顿，制造悬念 | `And this… is where it changes.` |
| `—` | 破折号，语气转折或强调 | `But—you have to do it manually.` |
| `—` 结尾 | 引出列表或下一句 | `Let me show you how most models work today—` |
| 单独成行的 `But—` | 强转折停顿 | `But—` / `👉 none of them can improve it.` |

---

## 三、👉 的用法

**👉 只用于关键爆点，不滥用。**

- 段落的核心结论
- 让观众记住的那一句
- 对比的胜负揭晓

```
✅ 👉 It fixes the system behind the problem.
✅ 👉 it evolves.
✅ 👉 MiniMax is Chef B.

❌ 👉 Today we'll cover three things:   ← 不是爆点，不用
```

---

## 四、修辞问句（Rhetorical Questions）

**用问句引导观众思考，自己接着回答。**

```
But then?
You run the same task again…
👉 Same error.

Why?
Because the system didn't actually learn anything.

So next time?
The problem doesn't come back.

If you want it better?
You update it. Manually.
```

**问句要短，1-4 个词最佳。**

---

## 五、常用开场句式

| 句式 | 用途 |
|------|------|
| `Alright—so today,` | 视频开头，自然过渡 |
| `Let me show you...` | 引入演示或解释 |
| `Let me give you a simple analogy.` | 引入类比 |
| `Now here's the problem.` | 引入痛点 |
| `Now here's the catch:` | 引入反转或限制 |
| `So this brings us to...` | 引入新概念 |
| `And this… is where X changes the game.` | 引入核心差异 |
| `In other words—` | 总结升华 |
| `From that point on?` | 描述结果 |

---

## 六、对比句型

**"But MiniMax?"系列——先提对手，再反转：**

```
Most AI tools today are reactive.
Something breaks → they fix it.

But MiniMax?
It doesn't just fix the problem.
👉 It fixes the system behind the problem.
```

**"Not just X… Y"结构：**

```
It doesn't just fix problems.
👉 It upgrades the system itself.
```

**"→" 箭头表示因果流：**

```
Something breaks → they fix it.
It breaks again → they fix it again.
```

---

## 七、禁止使用的写法

```
❌ "In conclusion, we can see that..."
❌ "It is worth noting that..."
❌ "Furthermore, additionally, moreover"
❌ 超过 15 个词的单句
❌ 被动语态（"it was found that..."）
❌ 多个形容词堆叠
❌ 学术/书面感词汇
```

---

## 八、节奏检查

生成脚本后，大声读一遍：

- [ ] 每句话是否自然停顿？
- [ ] 有没有需要换气的超长句？
- [ ] 👉 出现频率是否合适（每段 1 次为宜）？
- [ ] 问句是否自然，不像考试题？
- [ ] 对比是否清晰，听众一听就懂？
