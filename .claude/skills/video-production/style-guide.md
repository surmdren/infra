## 一、脚本结构

| 段落 | 时长 | 内容 |
|------|------|------|
| **Hook** | 0–30s | 结果先行，直接展示 AI 自动化最终产物 |
| **The Gap** | 30s–2min | 锚定痛点，描述手动操作有多慢多贵 |
| **Build** | 核心部分 | 环境准备 → API配置 → 逻辑构建 → 运行测试 |
| **CTA** | 结尾 | 引导下载 Prompt 模板或加入社群 |

### 8 步详细内容流程

```
① Hook       — 冲突/结果先行，0–30s，让人决定继续看
② 痛点        — 重复问题场景，建立共鸣（"你有没有遇过..."）
③ 概念        — 定义核心术语，降低认知门槛
④ 核心差异    — 爆点，只有 1 个，说清楚"这个和以前不一样在哪"
⑤ 类比        — 用已知解释未知，降低理解成本（"就像..."）
⑥ Trade-off  — 提升思考深度，诚实说出局限（"但它不适合..."）
⑦ 观点        — 你的立场，1 句话（"我认为..."）
⑧ CTA        — 引导互动，具体可执行（"在评论区告诉我..."）
```

> 四段式（Hook/Gap/Build/CTA）是时间分配框架，8 步是内容逻辑框架，两者叠加使用。

## 二、主题色规范

| 用途 | 颜色 | Hex |
|------|------|-----|
| 背景 | 深灰/黑 | `#0f0f0f` 或 `#1a1a2e` |
| 强调色1 | Cyan | `#00d4ff` |
| 强调色2 | Magenta | `#ff00ff` |
| 正文 | 白色 | `#ffffff` |
| 成功状态 | 绿色 | `#00ff88` |
| 错误状态 | 红色 | `#ff4444` |
| 次要文字 | 浅灰 | `#cccccc` |

**色彩比例法则（所有画面统一遵守）：**
- 背景：70%（深灰/黑）
- 白色正文：20%
- Cyan + Magenta 点缀：10%（不能大面积出现）

**语义颜色（每个颜色必须表达含义，不只是装饰）：**

| 含义 | 颜色 | Hex | 使用场景 |
|------|------|-----|---------|
| 问题 / 失败 / Before | 红色 | `#ff4444` | 演示痛点、错误状态、对比左侧 |
| 执行中 / 过程 | Cyan | `#00d4ff` | 工具运行中、Agent 处理中 |
| 成功 / 结果 / After | 绿色 | `#00ff88` | 最终输出、成功状态、对比右侧 |
| 输出节点 / 高亮 | Magenta | `#ff00ff` | 架构图输出、关键差异点 |

> 颜色用错比不用更糟：红色只能表示「问题」，绿色只能表示「成功」，不能随意用来装饰。

**架构图节点颜色约定（Eraser.io / Mermaid / HTML 全部统一）：**
- Cyan `#00d4ff` = 输入节点
- Magenta `#ff00ff` = 输出节点
- 白色 = 处理/中间步骤
- 绿色 `#00ff88` = 成功路径
- 红色 `#ff4444` = 错误路径

**避坑：**
- ❌ Cyan 和 Magenta 不要同时大面积出现
- ❌ 不要在深色背景上用深色文字
- ❌ 封面不要超过 3 种颜色

## 四、视觉元素规范

### 标志性效果
- **霓虹发光**：文字/图标周围加 Cyan/Magenta 光晕（`text-shadow` / `box-shadow`）
- **毛玻璃**：`backdrop-filter: blur(10px)` + 半透明背景（UI 窗口类）
- **文字动效**：淡入 + 轻微位移，不过度

### 单屏单信息原则（重要）

每一屏画面只表达**一个核心信息**：
- ✅ 一个数字、一个结论、一个对比
- ❌ 不要在同一屏同时展示多个 KPI、多个要点
- **对比结构是例外**：Before vs After 可以放一屏，但左右各只有一个焦点

**三种标准视觉模板：**

| 模板 | 结构 | 适用场景 |
|------|------|---------|
| 对比型 | 左 Before ❌ / 右 After ✅ | MiniMax vs Claude、手动 vs 自动 |
| 单点爆破型 | 大标题 → 核心视觉 → 关键词 | 评分、成本数字、速度数据 |
| 流程型 | 问题 → 执行 → 结果 | AI Agent 工作流演示 |

### 动态标注
- 箭头颜色：白色或 Cyan
- 高亮框：Cyan 边框 + 轻微背景色
- 触发时机：解释 AI 工具使用、工作流节点、数据关键点

### Smooth Zoom 触发时机（固定规则）
- 输入 API Key 时
- 关键代码行出现时
- 重要数字/结果出现时（如评分、成本、时间）
- Excel Sheet 切换时

## 六、转场 & 剪辑节奏

- **转场方式**：默认硬切；场景切换时用 0.3s 淡入淡出
- **平均镜头时长**：技术演示 8–15s，讲解段 3–8s
- **节奏检查**：1.5x 速度完整看一遍，标出无聊点剪掉
- **保留有价值的失败**：Prompt Tuning 修复报错的过程，观众最想学
- **前 30 秒必须出现视频最精彩画面**（Pattern Interrupt）

## 七、B-roll 生成规范（LibTV）

所有 LibTV 生成的 B-roll 必须使用以下 prompt 模板，保证风格统一：

### 标准 prompt 模板
```
[内容描述]，深色背景 #0f0f0f，Cyan #00d4ff 数字/元素发光效果，
科技感粒子特效点缀，16:9 比例，5-8秒时长，无人物出镜
```

### 常用 B-roll 类型及 prompt

**计数器/数字类：**
```
[数字内容] 逐步滚动动画，深色背景 #0f0f0f，Cyan #00d4ff 发光数字，
科技感粒子特效，16:9，5秒
```

**数据流/代码类：**
```
AI 代码高速滚动，多维度数据流扫描感，深蓝黑背景 #1a1a2e，
Cyan 文字发光，粒子特效，16:9，5秒
```

**转化/流程类：**
```
[图标A] 进入漏斗 → [图标B] 弹出的转化动画，深色背景，
Cyan 发光效果，科技感，16:9，5秒
```

**对比/分屏类：**
```
左右分屏对比，左侧 [内容A] 右侧 [内容B]，Cyan 分割线，
深色背景，计时器显示，16:9，8秒
```

## 八、屏幕录制标准（VS Code / OBS）

### VS Code 录屏 Profile
- **主题**：One Dark Pro（或同类深色主题）
- **字体**：JetBrains Mono，字号 18–20px
- **隐藏**：无关文件夹、扩展面板、不相关标签页
- **终端**：放底部，字号 16px
- **只保留**：当前演示用的文件和终端

### OBS 设置
- **分辨率**：1920×1080，60fps
- **录制格式**：MP4
- **场景**：全屏录制 + 右下角 PiP 圆形小窗
- **测试**：录制前 10 秒检查无杂音、无逆光

### `.env` 演示规范
- 永远演示 `.env` 配置方式（体现工程能力）
- API Key 用 `sk-xxx...xxx` 脱敏展示
- Smooth Zoom 放大 API Key 输入处

## 十四、安全区域（Safe Zone）

所有文字、字幕、标注必须在安全区域内，避免被平台 UI 或边框裁切：

```
上下左右各留 5% 边距
1920×1080 对应：水平安全区 96px，垂直安全区 54px
字幕底部距屏幕底边：至少 80px（避免被 YouTube 进度条遮挡）
```

**规则：**
- ❌ 不要把任何信息放到画面四周 5% 区域内
- ❌ 字幕不能贴底，留够 80px
- ✅ 动态标注箭头如果指向边缘元素，箭头本身必须在安全区内

---

## 十五、间距系统（8px Grid）

所有 HTML 视觉组件和字幕标注使用 **8px 基础网格**：

| 层级 | 间距 | 用途 |
|------|------|------|
| XS | 4px | 图标与文字间距 |
| S | 8px | 同组内元素间距 |
| M | 16px | 不同内容块间距 |
| L | 24px | 卡片内 padding |
| XL | 40px | 主要区块间距 |
| XXL | 64px | 页面级间距 |

Tailwind 对应：`gap-1`(4px) / `gap-2`(8px) / `gap-4`(16px) / `gap-6`(24px) / `gap-10`(40px) / `gap-16`(64px)

## 十六、图标规范

**锁定图标集：只用 Lucide Icons**

- ❌ 禁止混用其他图标库（Font Awesome、Material Icons、Heroicons 等）
- ✅ 所有 HTML 组件、演示截图标注、流程图图标统一用 Lucide
- **CDN**：`https://unpkg.com/lucide@latest`
- **尺寸规范**：
  - 正文旁图标：`w-4 h-4`（16px）
  - 卡片标题图标：`w-5 h-5`（20px）
  - 强调/展示图标：`w-8 h-8` 或 `w-10 h-10`
- **颜色**：跟随所在元素的强调色（Cyan / Magenta / 白色 / 状态色）

## 十七、字体克制原则

**同一画面最多使用 2 种字体**

| 场景 | 允许组合 |
|------|---------|
| 封面 | Poppins（标题）+ 无（只用一种）|
| HTML 视觉组件 | Poppins（标题）+ Inter（正文）|
| 代码演示画面 | Inter（字幕）+ JetBrains Mono（代码）|
| 手写风格组件 | Caveat（标题）+ Inter（正文）|

- ❌ 不要在同一画面同时出现 Poppins + Caveat + Inter
- ❌ 不要用超过 3 种字重（如只用 Regular + Bold，不加 Medium）

## 十八、Lower Third（下三分之一字幕条）

用于介绍工具名称、数据来源、人名等辅助信息。

**标准模板：**
```
位置：画面左下角，距底部 80px（安全区内）
背景：半透明深色 rgba(0,0,0,0.7) + 左侧 3px Cyan 竖线
主文字：Inter Bold，18px，白色
副文字：Inter Regular，14px，灰色 #cccccc
出现方式：从左侧滑入，0.3s
消失方式：淡出，0.2s
```

**示例：**
```
▌ MiniMax M2.7          ← 主文字（工具名/人名）
  platform.minimaxi.com  ← 副文字（来源/职位/价格）
```

**触发时机：**
- 第一次出现某个 AI 工具时
- 引用数据/价格时标注来源
- 对比段落切换工具时

## 十九、留白原则

留白是设计的一部分，不是浪费空间。

- **卡片内容不要超过 70% 面积**，留 30% 呼吸空间
- **标题与内容之间**：至少 16px 间距
- **相关内容靠近，不相关内容分开**（亲密性原则）
  - 工具名和价格放在一起
  - 不同演示场景之间明确分隔
- **画面中不要同时出现超过 5 个信息点**：超过时拆分为多个画面

## 二十一、节奏系统（Pacing System）

节奏是视频留存率的隐形杀手——信息密度均匀 = 无聊。

### 整体节奏公式

```
慢（Hook，建立悬念）→ 快（核心演示，信息密集）→ 慢（收尾，让信息沉淀）
```

### 关键停顿规则（剪辑时必须执行）

| 时机 | 停顿时长 | 原因 |
|------|---------|------|
| 反转 / 爆点 **之前** | **0.5 秒** | 给观众大脑一个"等等，什么？"的空间 |
| 金句 / 标志句式说完**之后** | **1 秒** | 让句子在脑子里回响，形成记忆 |
| 切换场景 / 演示工具时 | 0.3 秒淡入淡出 | 避免跳切带来的混乱感 |

### 节奏检查（剪辑完成后）

- [ ] 1.5x 速度完整看一遍，标出「感到无聊」的片段，果断剪掉
- [ ] 确认前 30 秒内出现视频最精彩的画面（Pattern Interrupt）
- [ ] 每隔 2–3 分钟有一个「节点」提示观众接下来会发生什么

## 二十二、每条视频制作 Checklist

> 发布前必须通过所有检查。这是对观众时间的承诺。

### 内容层
- [ ] 有反转（Before vs After，或出乎意料的结论）
- [ ] 有对比（数字/速度/成本的量化对比，不只是文字描述）
- [ ] 有金句（至少 1 个标志句式，重复 3 次以上）
- [ ] 核心差异只有 1 个（不要同时强调 5 个优点）

### 视觉层
- [ ] 语义颜色使用正确（红=问题，绿=成功，不混用）
- [ ] 有对比结构画面（Before/After 或流程型 ERROR→FIX→EVOLVE）
- [ ] 单屏单信息（没有一屏塞多个 KPI 或要点）
- [ ] 封面只用 Cyan 单强调色，无 Magenta

### 节奏层
- [ ] 反转前有 0.5s 停顿
- [ ] 金句后有 1s 停顿
- [ ] 整体节奏：慢→快→慢
- [ ] 1.5x 速度检查已完成，无聊片段已剪掉
