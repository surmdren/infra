# YouTube 视频脚本生成器

根据主题和受众，生成可直接录制的完整视频脚本。

## 语言要求

**所有生成内容必须使用英文**，包括：
- 口播脚本（Part 1/2/5）
- 标题候选
- YouTube 描述 + Tags
- 封面文案

以下内容使用中文：
- 制作注释（画面类型、素材说明、🎥📺📝 标注）
- 素材待办清单
- 本 SKILL 内部的工作流程说明

## 口语风格指南（必读）

生成脚本前，优先读取：
```
../video-production/oral-style-guide.md
```

该文件定义了频道的 native speaker 口播风格——短句、破折号停顿、👉 强调、修辞问句等。**所有脚本必须严格遵循此风格**，听起来像人说的，不像人写的。

如果找不到该文件，遵循以下核心原则：
- 一个想法一行，极短句
- 用 `…` 和 `—` 标记停顿
- 👉 只用于关键爆点
- 多用修辞问句（Why? But then? So next time?）
- 禁止：In conclusion / Furthermore / It is worth noting

## 微信文章输入模式

当用户提供 `mp.weixin.qq.com` URL 或粘贴了中文文章内容时，先提取正文，再进入正常流程。

### Step 0A：提取微信文章正文

**如果是 URL**，按优先级尝试：

**方法1 — CDP Proxy（优先）：**
```bash
curl -s http://localhost:3456/targets
```
找到匹配 URL 的 `targetId`，提取正文：
```bash
curl -s -X POST "http://localhost:3456/eval?target=<targetId>" \
  -d 'document.querySelector("#js_content")?.innerText || document.body?.innerText'
```

**方法2 — 请用户粘贴（备选）：**
若 CDP Proxy 不通，告知用户："微信文章需要登录，请复制全文粘贴给我"。

**如果是粘贴的内容**：直接使用。

### Step 0B：从文章提炼视频主题

提取正文后，分析文章并整理出：
- **视频主题**：文章最适合做成什么角度的视频（不是简单复述，而是找到对观众最有价值的切入点）
- **核心观点**：3-5 个值得展开的论点
- **潜在 Hook**：文章里最反常识或最有冲击力的一句话

将以上作为输入，替代用户手动填写主题，然后**直接进入 Step 1**（生成标题），不需要再问用户主题是什么。

## 输入

用户需提供（不足时主动询问）：

| 信息 | 说明 | 示例 |
|------|------|------|
| **视频主题** | 这条视频讲什么 | "如何用 AI 工具提升工作效率" |
| **目标受众** | 谁在看这个视频 | "25-35岁职场白领，有一定技术基础" |
| **产品/服务**（可选）| 是否推广某产品 | "我们的 SaaS 工具 XX" |
| **视频时长目标** | 大概多长 | "8-12 分钟" |
| **风格参考**（可选）| 参考哪个博主的风格 | "轻松幽默，像 XX 频道那种" |
| **研究报告**（可选）| 是否有 youtube-creator-research 的输出 | 路径或粘贴内容 |

如果用户只说了主题，先生成草稿，然后问是否需要调整受众定位和风格。

## 工作流程

### Step 1：生成 10 个候选标题

每种公式出 1-2 个，覆盖以下类型：

```
数字型：   [数字] 个 [关键词]，让你 [结果]
问题型：   为什么你的 [X] 总是 [失败/不够好]？
结果型：   我用 [XX 天/XX 方法] 实现了 [具体成就]
对比型：   [选项A] vs [选项B]：[行业] 从业者的真实体验
秘密型：   [行业] 高手都在用的 [XX] 方法（但很少人知道）
教程型：   [X] 分钟完整教程：[从零到结果]
反常型：   停止 [常见做法]！你一直在做错的 [XX]
紧迫型：   [时间] 内掌握 [技能]，否则你会落后
```

**标题写作原则：**
- 前 60 个字符最关键（移动端搜索结果显示限制）
- 包含核心关键词（帮助 SEO）
- 制造信息差或情绪张力
- 避免标题党（承诺内容必须兑现）

列出 10 个后，**标注推荐的前 3 个**并说明理由。

### Step 2：制定视频大纲

根据目标时长，分配各段时间：

**标准结构（8-12 分钟视频）：**

```
[0:00-0:20]  Demo Preview — 先展示结果画面（10-20秒），让观众看到终态
[0:20-0:40]  Hook — 制造悬念，"这是怎么做到的？"
[0:40-1:30]  问题铺垫 — 建立共鸣，为什么这个话题重要
[1:30-2:00]  预告 — "在这个视频里，你将学到..."
[2:00-7:00]  核心内容 — 主体部分（3-4个模块）
[7:00-8:00]  总结 + 行动建议
[8:00-8:30]  CTA — 订阅/点赞/跳转链接
```

**短视频结构（3-5 分钟）：**
```
[0:00-0:15]  Demo Preview — 结果先行
[0:15-0:30]  Hook
[0:30-0:50]  问题
[0:50-3:30]  核心内容（2-3个要点）
[3:30-4:00]  总结 + CTA
```

**⚠️ 强制规则：**
- **Demo Preview 必须在 90 秒内出现**，哪怕只有 5-10 秒的结果预览。先让观众看到"能做什么"，再解释"怎么做"。观众在决定继续看之前，需要先看到终点。
- **每 90 秒设置一个 Pattern Interrupt**：切换画面角度、抛一个新问题、或说一句反常识的话。在大纲中用 `[⚡PI]` 标注这些节点。

输出带时间戳的大纲，每个模块说明**要讲什么**和**为什么放在这里**。

### Step 3：撰写完整逐字脚本

按大纲逐段写出完整脚本。

**Hook 写法（最重要，决定留存率）：**

Hook 的 4 种有效开场方式：
1. **结果展示**：先展示最终成果，"你看到的这个，是我用 XX 方法实现的..."
2. **强烈问题**："如果你也有这个问题，这个视频可能会改变你的做法"
3. **反常识陈述**："大多数人都认为 X 是对的，但数据告诉我们完全不是"
4. **具体数字**："在过去 90 天里，我用这个方法让 [指标] 提升了 XX%"

**脚本格式：**
```
[时间戳] 模块名称
---
[旁白/画外音内容]

[场景注释：此处切换到屏幕录制 / 展示图表 / 切换 B-roll]

[关键词高亮：加粗展示核心概念]
---
```

**写脚本时注意：**
- 口语化，像在和朋友说话，不是在念文章
- **One-liner 核心观点**：每个视频必须提炼出一句话的核心金句（≤15词），这句话要在脚本中出现**两次**：第一次在中间内容高潮处，第二次在结尾前作为总结。这是观众会截图转发的那句话。
- **Pattern Interrupt**：每 90 秒必须有一个注意力重置点，用 `[⚡PI]` 标注。可以是：切画面、抛问题、反常识陈述、节奏突变。
- 每 2-3 分钟设置一个"节点"，提示观众接下来会讲什么（维持期待感）
- 在介绍产品/服务时，先铺垫痛点，再自然引出解决方案，避免硬广感
- 结尾 CTA 要具体："如果这个方法对你有用，点击下方订阅，我每周更新..."

### Step 4：生成配套内容

**封面文案（3 个方案）：**
- 封面文字通常 3-6 个字，放大关键词
- 配合标题形成"封面+标题"组合拳

```
方案 A：[关键词] + [数字/图标]
方案 B：[结果/状态] + [对比色]
方案 C：[问题/疑问句] 短版
```

**YouTube 描述模板：**
```
[视频核心内容 2-3 句话摘要，包含关键词]

在这个视频中，你将学到：
✅ [要点 1]
✅ [要点 2]
✅ [要点 3]

[时间戳导航]
0:00 介绍
X:XX [模块名]
...

[相关资源链接区]
👉 [产品/工具链接]
👉 [相关视频推荐]

[关注/订阅引导]

#标签1 #标签2 #标签3（5-10个相关标签）
```

**Tags 建议：**
列出 10-15 个 YouTube 标签（混合宽泛标签和长尾标签）

### Step 5：素材分配分析

逐段分析脚本，为每个段落标注对应的素材类型：

**判断规则：**
- 观众需要**读懂文字/结构** → `[HTML]` html-visual-generator
- 观众需要**感受氛围/场景** → `[Seedream]` seedream skill
- 需要**动态循环画面** → `[LibTV]` libtv-skill
- 需要**真实操作演示** → `[屏幕录制]`
- 需要**真人讲解** → `[真人出镜]`

输出格式：
```
| 段落 | 口播内容摘要 | 素材类型 | 具体描述 |
|------|------------|---------|---------|
| Hook | ... | LibTV + 真人出镜 | 动态 B-roll + 真人抛问题 |
| 痛点 | ... | HTML | 循环错误流程图 |
| 类比 | ... | Seedream | 混乱厨房 vs 高科技厨房 |
```

### Step 6：生成完整制作包（5 部分）

参考风格指南：`../video-production/style-guide.md`

**Part 1 — 完整口播脚本（带时间码）**
每段标注时间范围和时长，口语化、自然流畅。

**Part 2 — 提词器版（超精简 + 停顿）**
- 每行 ≤ 5 个词，下一行是上一行的自然延续
- 关键词放行首，不放行末（眼睛扫到行首就能抓住重点）
- 停顿标记遵循 oral-style-guide：`…` 口头停顿 / `—` 破折号转折 / 单独成行的 `But—` 强转折
- 禁止用「/」作停顿标记
- 重音词全大写
- 修辞问句独立成行（Why? / But then? / So next time?）
- `[⚡PI]` 标注 Pattern Interrupt 节点，提醒拍摄时切换画面或节奏

**Part 3 — 画面与制作笔记**
每个段落标注：
- 🎥 镜头（真人出镜的拍摄方式）
- 📺 B-roll/素材（HTML/Seedream/LibTV 具体内容）
- 📝 关键词字幕

**Part 4 — 节奏与停顿设计**
- 每个段落的节奏（快/中/慢）
- 黄金停顿点（爆点前 0.5s，金句后 1s）

**Part 5 — 一体版（拍摄专用）**
将 Part 1 + Part 2 + Part 3 + Part 4 合并，每段同时显示：
- 完整口播文字
- 节奏/停顿标注
- 镜头提示（极简版）

### Step 7：输出汇总

将所有内容整理保存到：
```
creator-notes/[年月]/[期数]/[视频名]-production-pack.md
```

包含：
1. 推荐标题（Top 3 加星标）
2. 素材分配表
3. Part 1-5 完整制作包
4. 封面文案方案
5. YouTube 描述 + Tags
6. 素材待办清单（HTML/Seedream/LibTV 各需生成什么）

## 质量检查

生成完成后，对脚本做自检：

- [ ] **Demo Preview 在 90 秒内出现了吗？** 观众必须先看到终点
- [ ] **One-liner 金句提炼了吗？** ≤15词，在脚本中出现两次（中间 + 结尾）
- [ ] **每 90 秒有 Pattern Interrupt 吗？** 用 `[⚡PI]` 标注了几处？
- [ ] Hook 开场是从「结果」开始，而不是从「背景」开始？
- [ ] 是否有清晰的"为什么看这个视频"的理由？
- [ ] 内容是否兑现了标题的承诺？
- [ ] 每个段落之间是否有过渡？
- [ ] CTA 是否明确、具体？
- [ ] 口语化程度够吗？（大声读一遍，听起来自然吗？）
- [ ] 提词器每行 ≤ 5 个词，关键词在行首？
- [ ] 素材分配合理吗？（读 vs 感受判断正确？）
- [ ] 一体版能否直接拿去拍摄，不需要切换文档？
