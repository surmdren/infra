# Sync and PR

批量为所有 target 项目提交当前更改并创建 Pull Request。

## 功能说明

这个 skill 会自动执行以下操作：

1. 读取 `.claude/sync-targets.json` 中配置的所有 target 项目
2. 对每个 target 项目：
   - 检查是否有未提交的更改（git status）
   - 如果有更改，创建新分支
   - 提交所有更改（自动生成 commit message）
   - 推送到远程仓库
   - 使用 `gh` CLI 创建 Pull Request
3. 生成批量提交报告

## 使用方法

```bash
# 为所有 target 项目提交更改并创建 PR
/sync-and-pr

# 为特定 target 项目提交更改并创建 PR
/sync-and-pr mingkun ceo-office

# 自定义 commit message 和 PR 标题
/sync-and-pr --message "feat: add new feature" --title "Add new feature"
```

## 参数说明

- `$ARGUMENTS`（可选）：
  - 不传参数：处理所有 target 项目
  - 传入 alias：只处理指定的 target 项目
  - `--message <msg>` 或 `-m <msg>`：自定义 commit message（默认：根据更改自动生成）
  - `--title <title>`：自定义 PR 标题（默认：使用 commit message 的第一行）
  - `--body <description>`：自定义 PR 描述（默认：自动生成更改摘要）
  - `--draft`：创建 Draft PR
  - `--skip-commit`：只创建 PR，不创建新的 commit（使用现有未推送的 commits）

## 前置要求

1. 所有 target 项目都是 git 仓库
2. 已安装并配置 `gh` CLI（GitHub CLI）
3. 对所有 target 仓库有推送权限
4. 已配置 `.claude/sync-targets.json`

## Instructions

### Step 1: 解析参数

从 `$ARGUMENTS` 中提取：
- Target aliases（如果指定）
- PR 配置参数（title, body, draft）

### Step 2: 读取同步配置

读取 `/Users/rick.ren/github/dreamai/my-skills/.claude/sync-targets.json`，获取所有 target 项目信息。

如果用户指定了特定的 aliases，只处理这些项目。

### Step 3: 对每个 target 执行 commit 和 PR 创建

对每个 target 项目执行以下操作：

#### 3.1 检查项目状态

```bash
# 进入目标项目
cd {target_path}

# 检查 git 状态
git status

# 检查是否有未提交的更改
if [ -z "$(git status --porcelain)" ]; then
  echo "⚠️  {target_alias}: No changes to commit, skipping..."
  continue
fi

# 获取当前分支
CURRENT_BRANCH=$(git branch --show-current)
```

#### 3.2 创建分支并提交（如果不在功能分支上）

```bash
# 如果在 main/master 分支，创建新分支
if [ "$CURRENT_BRANCH" = "main" ] || [ "$CURRENT_BRANCH" = "master" ]; then
  BRANCH_NAME="feat/batch-commit-$(date +%Y%m%d-%H%M%S)"
  git checkout -b $BRANCH_NAME
  echo "✅ Created new branch: $BRANCH_NAME"
else
  BRANCH_NAME=$CURRENT_BRANCH
  echo "ℹ️  Using existing branch: $BRANCH_NAME"
fi

# 添加所有更改
git add -A

# 生成 commit message（如果用户没有指定）
if [ -z "{commit_message}" ]; then
  # 自动生成 commit message：分析 git status 和 git diff
  CHANGED_FILES=$(git diff --cached --name-status | wc -l | xargs)
  ADDED=$(git diff --cached --name-status | grep '^A' | wc -l | xargs)
  MODIFIED=$(git diff --cached --name-status | grep '^M' | wc -l | xargs)
  DELETED=$(git diff --cached --name-status | grep '^D' | wc -l | xargs)

  COMMIT_MSG="chore: update project files

- $ADDED files added
- $MODIFIED files modified
- $DELETED files deleted

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>"
else
  COMMIT_MSG="{commit_message}

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>"
fi

# 提交更改
git commit -m "$COMMIT_MSG"
echo "✅ Committed changes"
```

#### 3.3 推送并创建 PR

```bash
# 检查远程分支是否存在
if git ls-remote --exit-code --heads origin $BRANCH_NAME > /dev/null 2>&1; then
  # 远程分支存在，直接 push
  git push origin $BRANCH_NAME
else
  # 远程分支不存在，push 并设置 upstream
  git push -u origin $BRANCH_NAME
fi

echo "✅ Pushed to remote"

# 检查是否已有 PR
EXISTING_PR=$(gh pr list --head $BRANCH_NAME --json number --jq '.[0].number' 2>/dev/null)

if [ -n "$EXISTING_PR" ]; then
  echo "ℹ️  PR already exists: #$EXISTING_PR"
  echo "   $(gh pr view $EXISTING_PR --json url --jq '.url')"
else
  # 创建 PR
  PR_TITLE="{pr_title}"
  if [ -z "$PR_TITLE" ]; then
    # 使用 commit message 第一行作为 PR 标题
    PR_TITLE=$(git log -1 --pretty=%B | head -n 1)
  fi

  PR_BODY="{pr_body}"
  if [ -z "$PR_BODY" ]; then
    # 自动生成 PR 描述
    PR_BODY="## Changes Summary

$(git log origin/main..$BRANCH_NAME --pretty=format:'- %s' 2>/dev/null || git log --pretty=format:'- %s' -1)

## Files Changed

\`\`\`
$(git diff --stat origin/main...$BRANCH_NAME 2>/dev/null || git diff --stat HEAD~1...HEAD)
\`\`\`

🤖 Generated by [sync-and-pr skill](https://github.com/surmdren/my-skills)"
  fi

  gh pr create \
    --title "$PR_TITLE" \
    --body "$PR_BODY" \
    {draft_flag}

  PR_URL=$(gh pr view --json url --jq '.url')
  echo "✅ Created PR: $PR_URL"
fi
```

### Step 4: 生成批量提交报告

汇总所有 target 项目的处理结果：

```markdown
## 批量提交和 PR 创建报告

生成时间: {timestamp}

| 项目 | 分支 | PR | 提交信息 |

| ceo-office | feat/batch-commit-20260129-143022 | [#45](PR_URL) | chore: update project files |

### ℹ️ 已有 PR（未创建新的）

1. 在各项目中 review PR
2. 通过 CI/CD 检查
3. Merge PR

## 命令记录

```

## 错误处理

- 如果 target 目录不存在，跳过并记录警告
- 如果 git 操作失败，跳过该 target 并继续处理其他项目
- 如果 PR 创建失败，输出错误信息但继续处理
- 所有错误都记录在最终报告中

## 注意事项

1. **权限要求**：需要对所有 target 仓库有推送权限和创建 PR 的权限
2. **gh CLI**：确保已安装并认证 `gh` CLI
3. **分支策略**：
   - 如果当前在 main/master 分支，会自动创建新的功能分支
   - 如果已在功能分支，会直接在当前分支提交
4. **PR 去重**：如果分支已有 PR，不会重复创建，只会推送最新提交
5. **错误处理**：某个项目失败不会影响其他项目的处理
6. **建议工作流**：
   - 先在单个项目测试 `/sync-and-pr mingkun`
   - 确认无误后再批量执行 `/sync-and-pr`
7. **安全提示**：检查 git status 确保不会提交敏感文件（.env, credentials 等）
