# 多语言 / i18n 方案

> **默认要求：** 所有前端页面默认支持中文（zh）和英文（en）双语。

## 框架选型

| 框架 | i18n 方案 | 路由方式 |
|------|----------|----------|
| **Astro** | `[...lang]` 动态路由 + `ui.ts` | 路径前缀 `/zh/` |
| **Next.js** | `next-intl` 或内置 i18n routing | 路径前缀或子域名 |
| **Vue/Nuxt** | `vue-i18n` + `@nuxtjs/i18n` | 路径前缀 |
| **React SPA** | `react-i18next` | 路径前缀 |

## URL 策略

| 方式 | 示例 | SEO | 推荐度 |
|------|------|-----|--------|
| **路径前缀** | `example.com/zh/products` | 最优 | ⭐⭐⭐⭐⭐ |
| 子域名 | `zh.example.com` | 好 | ⭐⭐⭐ |
| 查询参数 | `?lang=zh` | 差 | ⭐ |

**推荐**：路径前缀，`/zh/...` 中文，`/en/...` 英文（或默认无前缀为英文）

## Geo-based 语言检测（Next.js middleware）

```typescript
// middleware.ts
import { NextRequest, NextResponse } from 'next/server'

const LOCALE_COOKIE = 'preferred-locale'

export function middleware(request: NextRequest) {
  const pathname = request.nextUrl.pathname
  if (pathname.startsWith('/zh') || pathname.startsWith('/en')) {
    return NextResponse.next()
  }
  const cookieLocale = request.cookies.get(LOCALE_COOKIE)?.value
  if (cookieLocale === 'zh' || cookieLocale === 'en') {
    return NextResponse.redirect(new URL(`/${cookieLocale}${pathname}`, request.url))
  }
  // Vercel 自动注入 request.geo
  const country = request.geo?.country ?? 'US'
  const locale = country === 'CN' ? 'zh' : 'en'
  return NextResponse.redirect(new URL(`/${locale}${pathname}`, request.url))
}

export const config = {
  matcher: ['/((?!api|_next|.*\\..*).*)'],
}
```

语言切换组件：
```typescript
function switchLocale(locale: 'zh' | 'en') {
  document.cookie = `preferred-locale=${locale}; path=/; max-age=31536000`
  window.location.href = `/${locale}${window.location.pathname.replace(/^\/(zh|en)/, '')}`
}
```

## 翻译文件组织

```typescript
// src/i18n/ui.ts
export const ui = {
  en: {
    'nav.home': 'Home',
    'nav.products': 'Products',
  },
  zh: {
    'nav.home': '首页',
    'nav.products': '产品中心',
  },
} as const
```

**原则**：
- 所有 UI 文案集中在翻译文件，按 namespace 分组
- 组件通过 `useTranslations()` 获取，不硬编码
- 组件从 URL 检测语言，不依赖 prop 传递

## SEO 多语言

```html
<link rel="alternate" hreflang="en" href="https://example.com/products" />
<link rel="alternate" hreflang="zh-CN" href="https://example.com/zh/products" />
<link rel="alternate" hreflang="x-default" href="https://example.com/products" />
```

## 数据库多语言设计

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **列模式** | `name_en`, `name_zh` 同表 | 语言少（2-3种）|
| **行模式** | `translations(id, locale, field, value)` | 语言多、字段多 |
| **JSON 模式** | `name: {"en": "...", "zh": "..."}` | PostgreSQL/MongoDB |
| **CMS 托管** | CMS 自动管理多 locale | 推荐（减少自建）|

## 常见反模式

- 组件内硬编码 `isZh ? '中文' : 'English'`
- 用 Cookie 存语言但不体现在 URL（SEO 灾难）
- 每种语言维护完全独立的页面文件
- 忘记翻译 meta title/description
