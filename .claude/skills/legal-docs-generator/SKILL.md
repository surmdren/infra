# Legal Docs Generator

为产品生成正式法律文件，直接可用，不是填空模板。

## 输出文件

```
legal/
├── privacy-policy.md       # 隐私政策
├── terms-of-service.md     # 用户协议
└── cookie-policy.md        # Cookie 政策
```

## Phase 0：信息收集

先扫描项目，自动提取能获取的信息，只询问无法推断的内容：

```bash
cat package.json 2>/dev/null | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('name',''), d.get('description',''))"
ls docs/ PRD/ TechSolution/ 2>/dev/null | head -20
```

收集后向用户确认以下关键信息：

```
公司名称：          DreamAI Ltd.
产品名称：          [产品名]
公司注册地：        [中国大陆 / 香港 / 新加坡 / 美国 / 其他]
目标用户市场：      [中国 / 欧盟 / 美国 / 全球]
联系邮箱：          legal@example.com
网站域名：          https://example.com
收集的数据类型：    [姓名/邮箱/手机号/支付信息/位置/行为数据 等]
是否有用户账号系统：[是/否]
是否处理支付：      [是/否，使用什么支付方式]
是否使用 Analytics：[GA4/PostHog/Mixpanel 等]
是否有第三方登录：  [Google/GitHub/微信 等]
数据存储地区：      [中国境内/境外/两者都有]
```

## Phase 1：生成隐私政策

根据目标市场选择合规框架：

| 市场 | 适用法规 | 核心要求 |
|------|---------|---------|
| 中国 | PIPL、数据安全法 | 数据本地化、单独同意、删除权 |
| 欧盟 | GDPR | 明确法律依据、数据主体权利 |
| 美国 | CCPA（加州）| 不出售个人信息、选择退出权 |
| 全球 | 以 GDPR 为基准 | 满足所有市场最严标准 |

**必须包含：**

1. 信息收集范围（逐项列明，不用"等"模糊表述）
2. 使用目的和法律依据（GDPR 要求明确 lawful basis）
3. 第三方共享清单（Analytics、支付、CDN 等，每个都列出）
4. 数据存储地区和跨境传输说明
5. 数据保留期限（每类数据单独说明）
6. 用户权利：查阅、更正、删除、导出、拒绝处理
7. Cookie 和追踪技术说明
8. 未成年人保护条款（13岁/16岁限制）
9. 政策更新通知机制
10. 联系方式和投诉渠道

**PIPL 额外要求（面向中国用户必须包含）：**
- 个人信息处理者身份和联系方式
- 境外传输的目的地国家及安全评估结论
- 单独同意事项清单（敏感信息、境外传输、第三方共享）
- 个人信息保护负责人联系方式

## Phase 2：生成用户协议

**必须包含：**

1. 服务描述和适用范围
2. 账号注册和使用规则
3. 用户行为规范（禁止事项明确列举）
4. 知识产权归属（用户内容 vs 平台内容）
5. 付费条款（如有）：价格、退款政策、自动续费说明
6. 服务中断和免责条款
7. 争议解决和管辖法院
8. 协议修改通知方式
9. 终止账号的条件和后果

**中国市场：** 管辖法院选中国境内，内容审核规则符合监管要求。

**海外市场：** 仲裁条款（AAA/ICC），管辖法律选择，集体诉讼豁免。

## Phase 3：生成 Cookie 政策

按类别分类列明所有 Cookie：

| 类别 | 示例 | 是否必要 | 保留期 |
|------|------|---------|--------|
| 必要 | 登录状态、CSRF token | 是 | Session/30天 |
| 功能 | 语言偏好、主题设置 | 否 | 1年 |
| 分析 | GA4、PostHog | 否 | 2年 |
| 营销 | 广告追踪、再营销 | 否 | 90天 |

每个第三方 Cookie 须列出提供商名称和其隐私政策链接，以及如何禁用的操作指引。

## Phase 4：输出检查

生成完成后自动检查，确保：

- [ ] 无未替换占位符（`[TODO]`、`[填写]`、`[公司名]` 等）
- [ ] 联系邮箱和网站域名已填入
- [ ] 数据类型与项目实际收集一致
- [ ] 生效日期已填写

## 文件格式

每个文件开头：
```markdown
# [文件标题]

**生效日期**：[日期]
**最后更新**：[日期]

---
```

语言：面向中国用户生成中文；面向海外用户生成英文；全球产品生成中英双语。
