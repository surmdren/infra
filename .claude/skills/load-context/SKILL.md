# 对话上下文加载器

## Overview

从 `.claude/context.md` 读取之前保存的对话上下文，快速恢复项目状态，继续之前的工作。

```
1. 读取 .claude/context.md 文件
2. 解析上下文内容
3. 向用户展示项目当前状态
4. 准备继续工作
```

## Instructions

你是一名【项目助理 + 技术文档员】，负责帮助用户快速恢复之前的工作状态。

### 工作流程

#### Step 1: 检查上下文文件

```bash
# 检查上下文文件是否存在
if [ -f ".claude/context.md" ]; then
  echo "找到上下文文件"
  cat .claude/context.md
else
  echo "未找到上下文文件，提示用户使用 /save-context 保存"
fi
```

#### Step 2: 读取并解析上下文

读取 `.claude/context.md` 的全部内容，提取关键信息：

1. **项目信息** - 工作目录、Git 仓库、当前分支
2. **项目背景** - 项目是什么，要解决什么问题
3. **当前任务** - 之前在做什么
4. **进度概览** - 已完成、进行中、待办事项
5. **技术栈** - 使用的技术
6. **重要决策** - 技术决策及原因
7. **文件变更** - 修改过的文件
8. **下次继续** - 建议的下一步行动

#### Step 3: 向用户展示状态

以简洁清晰的方式向用户展示项目当前状态，并询问是否继续之前的工作。

### 输出格式

```markdown
## 上下文已恢复

**项目**: {project_name}
**分支**: {current_branch}
**上次会话**: {timestamp}

### 当前任务
{current_task}

### 进度概览
**已完成**: {n} 项
**进行中**: {n} 项
**待办**: {n} 项

### 下次建议
1. {next_action_1}
2. {next_action_2}

---

是否继续上次的任务？或者有其他需要？
```

### 上下文文件不存在时

如果 `.claude/context.md` 不存在：

```markdown
未找到保存的上下文文件。

如果你之前使用过 `/save-context`，请确认：
1. 是否在正确的项目目录下
2. .claude/context.md 文件是否存在

如果你是新对话，可以从以下开始：
1. 告诉我你要做什么
2. 使用 `/save-context` 在工作后保存进度
```

## 配合使用

### 与 save-context 配合

```
工作流程:
1. 开始工作 → ...
2. 完成阶段性任务 → /save-context
3. 关闭对话
---
4. 新对话开始 → /load-context
5. 继续工作 → ...
6. 完成任务 → /save-context
```

### 自动加载

在 `CLAUDE.md` 中添加提示：

```markdown
## 新对话开始

如果你是新对话，建议先运行 `/load-context` 恢复之前的上下文。
```

## 最佳实践

1. **新对话优先** - 开始新对话时先加载上下文
2. **工作后保存** - 完成重要工作后使用 `/save-context`
3. **确认状态** - 加载后确认当前 Git 分支和文件状态
4. **快速定位** - 利用"关键代码位置"快速找到文件

## 适用场景

- 长期项目中断后恢复
- 每日开始工作时
- 项目交接
- 多设备同步工作
- 代码审查前了解背景
