# 竞品分析

## 概述

系统化的竞争情报与战略规划框架。通过应用成熟的商业分析框架，将原始竞品信息转化为可执行的战略建议。

## 分析框架

### 1. 竞品识别

首先明确分析对象：

**直接竞争对手** - 相同产品，相同市场
- 示例：网约车领域，滴滴 vs. 快的打车

**间接竞争对手** - 不同产品，解决相同需求
- 示例：网约车领域，滴滴 vs. 公共交通、私家车

**潜在进入者** - 尚未进入但有能力进入
- 示例：拥有相关能力的科技巨头

### 2. 分析维度

对每个竞品，从以下维度分析：

| 维度 | 关键问题 |
|------|----------|
| **产品** | 功能、质量、用户体验、创新速度 |
| **商业模式** | 收入来源、定价、单位经济、增长模型 |
| **市场** | 市场份额、增长率、客户细分、地域覆盖 |
| **地理** | 国内 vs 国外市场 | ⭐ 新增 |
| **能力** | 技术、团队、合作伙伴、供应链、品牌 |
| **财务** | 融资、收入、盈利能力、烧钱率 |
| **战略** | 定位、差异化、增长策略、并购活动 |

### 2.1 地理维度分析（新增）

**国内 vs 国外市场差异分析**

**国内市场特殊考虑**
- **政策环境**：监管政策变化快，需关注合规要求
- **平台生态**：微信、支付宝、抖音等平台依赖
- **用户习惯**：移动优先、社交强依赖
- **文化因素**：社交推荐、口碑传播、信任链

**国外市场特殊考虑**
- **法律法规**：GDPR、CCPA、GDPR 等合规要求
- **文化差异**：直接沟通 vs 委婉含蓄
- **支付方式**：信用卡 vs 电子钱包
- **数据隐私**：隐私意识强、cookie 法律严格

### 3. 分析框架

根据分析目标选择合适的框架：

**SWOT 分析**（快速概览）
```
优势：内部优势
劣势：内部劣势
机会：外部机会
威胁：外部威胁
```

**波特五力模型**（行业结构）
```
1. 供应商议价能力
2. 买家议价能力
3. 同业竞争强度
4. 新进入者威胁
5. 替代品威胁
```

**价值主张画布**（差异化分析）
```
客户工作 → 痛点 → 收益
产品 → 痛点缓解 → 收益创造
匹配度分析
```

**蓝海战略**（市场开拓）
```
ERCC 矩阵：
- 剔除
- 减少
- 提高
- 创造
```

## 数据来源

尽可能从以下渠道收集信息：

**公开信息源**
- 公司官网、博客、新闻稿
- 财报（针对上市公司）
- 应用商店、评论网站
- 社交媒体、论坛、社区
- 招聘信息（揭示优先级/方向）
- 专利申请

**竞争情报工具**

**国内市场工具**
- **数据平台**：艾瑞咨询、易观分析、QuestMobile
- **社交媒体数据**：微信指数、微博指数、抖音数据分析
- **电商情报**： 䧹妈妈、米读、数据观
- **移动分析**：QuestMobile、TalkingData、极光大数据
- **融资数据**：IT桔子、36氪、企查查

**国外市场工具**
- **数据平台**：SimilarWeb、SEMrush（网站流量）
- **应用分析**： App Annie、Sensor Tower、Apptopia
- **融资数据**：Crunchbase、PitchBook
- **职业社交**：LinkedIn（团队、招聘）
- **开发者工具**：GitHub（代码活跃度）、Stack Overflow（技术趋势）

**用户研究**
- 用户访谈（竞品客户）
- 问卷调查
- 可用性测试
- 评论分析（App Store、G2、Capterra）

## 最佳实践

1. **保持客观** - 承认竞品优势，不要低估
2. **保持时效** - 市场变化快，使用最新数据
3. **具体明确** - 使用数字、案例、截图
4. **聚焦可执行** - 每个洞察都应指导战略
5. **交叉验证** - 使用多个来源验证信息
6. **定期更新** - 竞争格局不断演变

## 常见陷阱

- ❌ 分析瘫痪 - 不要过度分析，尽快行动
- ❌ 功能清单竞赛 - 功能多 ≠ 产品好
- ❌ 忽视新兴竞争者 - 重大威胁常来自意想不到的地方
- ❌ 确认偏误 - 不要只寻找支持假设的数据
- ❌ 静态分析 - 市场和竞品变化快
