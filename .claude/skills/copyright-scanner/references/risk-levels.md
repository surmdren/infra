# 风险等级定义与处理建议

## 风险等级标准

### 🔴 高风险（需立即处理，阻断集成）

满足以下任一条件：

| 条件 | 示例 |
|------|------|
| 明确禁止商业使用的协议 | GPL、AGPL、CC-BY-NC、Proprietary |
| 检测到破解版特征码 | eval+base64混淆、商业系统特征 + 无合法授权证明 |
| 版权声明被刻意删除 | 代码注释中有 "copyright removed"、加密保护被破解 |
| 无任何 LICENSE 文件且无版权声明 | 默认 All Rights Reserved，不可使用 |
| 第三方公司版权声明 | 代码头部写着别家公司版权，来源不明 |

**处理方式**：
1. 停止集成，隔离该代码
2. 查找合规替代品
3. 如必须使用，向版权方购买商业授权
4. 通知法务团队评估风险

---

### 🟡 中风险（需确认后才可使用）

满足以下任一条件：

| 条件 | 示例 |
|------|------|
| 有传染性但可商用的协议 | LGPL、MPL-2.0 |
| 协议未知但代码来源可追溯 | 无 LICENSE 但 GitHub 可查到原仓库 |
| 需要署名的协议 | CC-BY-4.0 |
| 商业软件特征 + 有合法授权文件 | 微擎官方购买版 |
| 大量文件缺少版权声明头 | 规范问题，需补充 |

**处理方式**：
1. 确认使用场景符合协议要求
2. LGPL：确保动态链接，不要修改库本身
3. 无 LICENSE：联系原作者获取明确授权
4. 补充版权声明头

---

### 🟢 低风险（知悉即可，无需阻断）

满足以下条件：

| 条件 | 示例 |
|------|------|
| 宽松开源协议 | MIT、Apache-2.0、BSD、ISC、CC0 |
| 有完整的版权声明头 | Copyright 2024 xxx, MIT License |
| 无商业软件特征 | 纯净代码 |

**处理方式**：
1. 保留 LICENSE 文件和版权声明头
2. 如果 Apache-2.0，保留 NOTICE 文件（如有）
3. 记录到项目的开源协议清单（THIRD-PARTY-NOTICES.md）

---

## 特殊情况处理

### 代码无 LICENSE 文件
- **默认**：All Rights Reserved，版权归作者所有
- **处理**：
  - 如果是知名开源项目（有 GitHub），查看历史提交是否有过 LICENSE
  - 联系原作者请求授权
  - 不建议直接使用

### eval + base64 混淆代码
- 这是破解版商业软件的典型特征
- 混淆的目的是绕过授权检查
- **高风险**，不建议集成到任何商业产品

### 加密保护文件（ionCube/Zend）
- 合法商业软件可能使用加密保护
- 如果有正版授权证书（license.key 文件），风险降低
- 无授权证书的加密文件 = 破解版，**高风险**

### 双重授权（Dual License）
- 部分项目同时提供 GPL + 商业授权（如 MySQL、Qt）
- GPL 版本开源使用，商业版本付费
- 使用时需明确使用的是哪个版本

## 输出报告中的措辞规范

| 情况 | 推荐措辞 |
|------|---------|
| 高风险，明确禁止商用 | "⚠️ 该协议禁止商业使用，需替换或购买授权" |
| 高风险，破解版特征 | "⚠️ 检测到商业软件特征码，可能为破解版，存在法律风险" |
| 中风险，LGPL | "需确认使用方式为动态链接，不可修改库本身" |
| 中风险，无 LICENSE | "需联系原作者确认授权，或查找有明确协议的替代品" |
| 低风险，MIT | "协议兼容商业使用，请保留 LICENSE 文件" |
